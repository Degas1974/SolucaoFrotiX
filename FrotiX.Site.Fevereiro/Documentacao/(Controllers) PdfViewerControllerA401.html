PdfViewerController

Visualização de PDFs com Syncfusion • Última atualização: 08/01/2026 • Versão 2.0

Visão geral

Controller genérico para servir PDFs ao componente Syncfusion: carrega via caminho físico ou Base64, renderiza páginas e expõe bookmarks. Base para CNH, multas e demais documentos.

  • Suporte a documentos locais e payloads inline (Base64).
  • Integra com renderer Syncfusion server-side.
  • Compartilha lógica com PdfViewerCNH.
Endpoints
  • POST /api/PdfViewer/Load – { document, isFileName }.
  • POST /api/PdfViewer/Bookmarks – retorna bookmarks do PDF.
  • POST /api/PdfViewer/RenderPdfPages – renderiza páginas específicas.
Interconexões
  • Páginas que exibem PDFs diversos (relatórios, comprovantes, CNH/multas via variantes).
  • Renderer Syncfusion PDF.
Cuidados
  • Garantir permissões de leitura no caminho físico.
  • Base64 grande: considerar compressão ou links temporários.
  • Bookmarks vazios: verificar se o PDF possui outline.
