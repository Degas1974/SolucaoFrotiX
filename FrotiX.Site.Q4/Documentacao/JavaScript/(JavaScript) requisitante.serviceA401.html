Serviço de Cadastro Inline de Requisitantes na Agenda

Versão: 1.2
Última Atualização: 12/01/2026
Localização: wwwroot/js/agendamento/services/
Visão Geral

O arquivo requisitante.service.js é um **serviço JavaScript** fundamental que permite aos usuários criar novos requisitantes **durante** o processo de agendamento de viagens, sem precisar sair do modal principal e navegar para outra tela.

Este sistema de "cadastro inline" melhora drasticamente a experiência do usuário ao eliminar a necessidade de interromper o fluxo de trabalho. Quando o usuário está preenchendo um agendamento e percebe que o requisitante necessário não existe, ele pode criá-lo imediatamente através de um accordion que se abre no próprio modal, preenchê-lo rapidamente e continuar o agendamento sem perder contexto.

Importância Crítica: Este arquivo foi recentemente corrigido (12/01/2026) para resolver um bug grave onde o accordion fechava automaticamente ao selecionar o Setor no DropDownTree, limpando todos os campos preenchidos e impedindo o salvamento.
Características Principais
  • ✓ Cadastro Inline - Criar requisitante sem sair do modal de agendamento
  • ✓ Accordion Controlado - Sistema de abertura/fechamento manual via botão toggle
  • ✓ Proteção Robusta - MutationObserver + desabilitação Bootstrap Collapse
  • ✓ API Integration - Comunicação com backend via AJAX POST
  • ✓ Validações Client-Side - Campos obrigatórios com feedback visual
  • ✓ Auto-Atualização - Após salvar, atualiza dropdowns automaticamente
  • ✓ Captura de Setores - Reutiliza dados já carregados na página
  • ✓ Sincronização Syncfusion - Integra com DropDownTree, ComboBox, NumericTextBox
Tecnologias Utilizadas
Tecnologia Versão Uso
JavaScript ES6 - Linguagem base (IIFE, arrow functions, async/await)
jQuery 3.x AJAX calls, manipulação DOM
Syncfusion EJ2 - DropDownTree, ComboBox, NumericTextBox
Bootstrap 5 - Accordion UI (com desabilitação programática)
MutationObserver API - Monitoramento de alterações DOM
