Endpoints de Monitoramento e Gerenciamento

Endpoints da API (Continuação)

Resposta do POST /api/DocGenerator/generate:

{
  "success": true,
  "data": {
    "jobId": "doc-gen-2026-01-13-14-30-45-abc123",
    "startedAt": "2026-01-13T14:30:45.123Z",
    "totalFiles": 15
  }
}

O job é executado em background e envia atualizações de progresso via SignalR. O cliente pode monitorar o status usando o jobId retornado.

GET /api/DocGenerator/job/{jobId}

Propósito: Obtém o status detalhado de um job de geração em execução ou já finalizado.

Parâmetros: jobId (string, na URL)

Retorno (JSON):

{
  "success": true,
  "data": {
    "jobId": "doc-gen-2026-01-13-14-30-45-abc123",
    "startedAt": "2026-01-13T14:30:45.123Z",
    "completedAt": null,
    "isRunning": true,
    "isCancelled": false,
    "totalFiles": 15,
    "processedFiles": 7,
    "successCount": 6,
    "failedCount": 1,
    "skippedCount": 0,
    "currentFile": "Controllers/ViagemController.cs",
    "currentStatus": "Processing",
    "results": [...]
  }
}

Use este endpoint para implementar barras de progresso ou dashboards de monitoramento em tempo real.

POST /api/DocGenerator/job/{jobId}/cancel

Propósito: Cancela um job de geração em execução.

Parâmetros: jobId (string, na URL)

Retorno (JSON):

{
  "success": true,
  "message": "Job cancelado"
}

O cancelamento é cooperativo: o job para na próxima oportunidade segura, finalizando o arquivo atual antes de encerrar.

GET /api/DocGenerator/jobs?count=10

Propósito: Lista os jobs de geração mais recentes (histórico).

Parâmetros Query: count (int, opcional, padrão=10)

Retorno (JSON):

{
  "success": true,
  "data": [
    {
      "jobId": "doc-gen-2026-01-13-14-30-45-abc123",
      "startedAt": "2026-01-13T14:30:45.123Z",
      "completedAt": "2026-01-13T14:35:12.456Z",
      "isRunning": false,
      "totalFiles": 15,
      "successCount": 14,
      "failedCount": 1,
      "skippedCount": 0
    }
  ]
}

POST /api/DocGenerator/cache/clear

Propósito: Limpa todo o cache de documentação, forçando regeneração completa na próxima execução.

Retorno (JSON):

{
  "success": true,
  "message": "Cache limpo"
}

Use com cautela: limpar o cache pode aumentar significativamente o tempo da próxima geração.

GET /api/DocGenerator/tests/run

Propósito: Executa todos os testes unitários do sistema DocGenerator e retorna o relatório.

Retorno (JSON):

{
  "success": true,
  "data": {
    "report": "Todos os testes passaram (15/15)",
    "executedAt": "2026-01-13T14:45:30.789Z"
  }
}

Útil para diagnóstico e validação do sistema antes de executar gerações grandes.
