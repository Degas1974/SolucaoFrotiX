Program.cs — Inicialização do Host

Arquivo: Program.cs
Papel: ponto de entrada, criação do host e handlers globais.
Este arquivo garante o bootstrap seguro do FrotiX, com fallback de log em arquivo caso o DI falhe.

Responsabilidades Principais

  • Criar o host via CreateHostBuilder.
  • Registrar handlers globais para exceções.
  • Manter log de emergência quando não há DI.

Fluxo de Inicialização

// Build + Handlers
var host = CreateHostBuilder(args).Build();
ConfigureGlobalExceptionHandlers(host.Services);

// Inicia a aplicação
host.Run();

Tratamento de Erros

  • Todo o Main está envolvido por try/catch.
  • Erro crítico dispara Alerta.TratamentoErroComLinha.
  • Fallback grava log direto em arquivo.

Snippet: Handler Global

AppDomain.CurrentDomain.UnhandledException += (sender, e) =>
{
    logService.Error("ERRO NÃO TRATADO (AppDomain)",
        e.ExceptionObject as Exception,
        "AppDomain",
        "UnhandledException");
};
