Regras críticas
  • Não apagar os arquivos `.md`; criar `.html` correspondentes com sufixo A4 em ordem alfabética.
  • Atualizar documentação no mesmo ciclo de mudança de código (ver REGRA-CRITICA-ATUALIZACAO-DOCUMENTACAO.md).
  • Respeitar visual FrotiX: header laranja, borda preta + branca, fundo cinza-claro, cards brancos, snippets em azul petróleo.
  • Usar ícones duotone abundantes, mas sem poluição; cada card com ícone temático.
