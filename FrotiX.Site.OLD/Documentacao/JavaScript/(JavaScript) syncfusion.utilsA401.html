Utilitários e Configurações Globais para Componentes Syncfusion EJ2

Versão: 1.1
Última Atualização: 12/01/2026
Localização: wwwroot/js/agendamento/utils/
Visão Geral

O arquivo syncfusion.utils.js é o coração da configuração dos componentes Syncfusion EJ2 no sistema FrotiX. Ele centraliza todas as funções utilitárias e configurações de localização (pt-BR) necessárias para que os componentes funcionem corretamente e exibam textos, datas e números no formato brasileiro.

Sem este arquivo, os componentes Syncfusion exibiriam textos em inglês, usariam formato de data americano (MM/DD/YYYY) e gerariam erros ao tentar formatar números e porcentagens por falta de dados de localização.

Importância Crítica: Este arquivo deve ser carregado ANTES da criação de qualquer componente Syncfusion. A função configurarLocalizacaoSyncfusion() deve ser chamada no início da página para garantir localização correta.
Características Principais
  • ✓ Configuração completa de localização pt-BR (textos, calendário, números)
  • ✓ Funções utilitárias para obter instâncias e valores de componentes
  • ✓ Gerenciamento global de tooltips (Syncfusion e Bootstrap)
  • ✓ Configuração de paste de imagens em RichTextEditor
  • ✓ Dados CLDR completos (calendário + formatação numérica)

Funções Utilitárias Principais

1. getSyncfusionInstance(id)

Propósito: Obtém a instância Syncfusion de um elemento HTML pelo ID.

Uso: Permite acessar métodos e propriedades de componentes Syncfusion de forma segura.

// Exemplo de uso
const veiculoCombo = window.getSyncfusionInstance('lstVeiculo');
if (veiculoCombo) {
    console.log('Valor selecionado:', veiculoCombo.value);
    veiculoCombo.dataBind(); // Atualizar componente
}

2. getSfValue0(inst)

Propósito: Obtém o primeiro valor de um componente Syncfusion (útil para multi-seleção).

Uso: Quando um componente pode retornar array ou valor único, esta função normaliza para sempre retornar o primeiro valor.

// Exemplo de uso
const motorista = window.getSfValue0(motoristaCombo);
console.log('ID do motorista:', motorista); // Sempre valor escalar ou null

3. limpaTooltipsGlobais(timeout)

Propósito: Remove todos os tooltips (Syncfusion e Bootstrap) para evitar duplicações.

Uso: Chamar ao fechar modais ou trocar de página para limpar tooltips órfãos.

// Exemplo de uso - ao fechar modal
$('#modalViagens').on('hidden.bs.modal', function() {
    window.limpaTooltipsGlobais(200); // Aguarda 200ms e limpa
});

4. configurarLocalizacaoSyncfusion()

Propósito: Configura localização completa pt-BR para TODOS os componentes Syncfusion.

Uso: CRÍTICO Chamar na inicialização da página, ANTES de criar componentes.

// Exemplo de uso - no início do main.js
$(document).ready(function() {
    // 1. Configurar localização PRIMEIRO
    window.configurarLocalizacaoSyncfusion();

    // 2. Depois criar componentes
    new ej.calendars.DatePicker({
        // ... configurações
    }).appendTo('#txtDataInicial');
});
Configuração CLDR pt-BR

O CLDR (Common Locale Data Repository) fornece dados essenciais para localização. O objeto ptBRCldr contém três seções fundamentais:

1. Identity (Identificação)

Define a versão do CLDR e idioma base.

"identity": {
    "version": { "_cldrVersion": "36" },
    "language": "pt"
}

2. Dates (Calendário)

Traduz meses e dias da semana para português:

  • Meses abreviados: jan, fev, mar, abr, mai, jun, jul, ago, set, out, nov, dez
  • Meses completos: janeiro, fevereiro, março, abril, maio, junho, julho, agosto, setembro, outubro, novembro, dezembro
  • Dias abreviados: dom, seg, ter, qua, qui, sex, sáb
  • Dias completos: domingo, segunda, terça, quarta, quinta, sexta, sábado

3. Numbers (Formatação Numérica) ⚠️ CRÍTICO

Adicionado em 12/01/2026 (versão 1.1)
Esta seção foi adicionada para corrigir erro "Cannot read properties of undefined (reading 'percentSign')" que ocorria ao interagir com componentes Syncfusion.

Símbolos numéricos pt-BR:

  • decimal: "," (vírgula como separador decimal → 1,5)
  • group: "." (ponto como separador de milhar → 1.000)
  • percentSign: "%" (símbolo de porcentagem)
  • plusSign: "+" / minusSign: "-"
  • infinity: "∞" / nan: "NaN"

Formatos:

  • Números decimais: #,##0.### → 1.234,567
  • Porcentagens: #,##0% → 75%
  • Moedas: ¤ #,##0.00 → R$ 1.234,56

Moedas suportadas:

  • BRL: Real brasileiro (R$)
  • USD: Dólar americano (US$)
  • EUR: Euro (€)
