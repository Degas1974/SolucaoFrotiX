Problema Resolvido

Correção Crítica do Fechamento Automático do Accordion

Problema Original
Sintoma Crítico: Usuário não conseguia cadastrar requisitante durante agendamento.

Fluxo do Problema

  1. Usuário clicava em "Novo Requisitante" (accordion abria)
  2. Preenchia todos os campos: Ponto, Nome, Ramal, Email
  3. Clicava no DropDownTree de Setor para selecionar um setor
  4. ❌ BUG Accordion fechava automaticamente
  5. ❌ BUG Todos os campos eram limpos
  6. ❌ BUG Usuário não conseguia salvar o requisitante

Impacto no Usuário

  • Frustração extrema ao perder dados preenchidos
  • Necessidade de tentar múltiplas vezes sem sucesso
  • Interrupção completa do fluxo de trabalho de agendamento
  • Possível necessidade de navegar para outra tela para criar requisitante
Causa Raiz Identificada

Bootstrap Collapse Automático

O accordion de requisitante utiliza Bootstrap 5 com a classe accordion-collapse collapse show. O Bootstrap Collapse possui um comportamento automático de fechar o accordion quando detecta cliques "fora" do elemento accordion.

Popup Syncfusion Fora do Accordion

O DropDownTree do Syncfusion, quando aberto, renderiza o popup (lista de setores) diretamente no <body>, e não dentro da estrutura DOM do accordion. Isso significa que o popup tecnicamente está "fora" do accordion.

Diagrama do Problema

<div id="accordionRequisitante"> <div id="collapseRequisitante" class="accordion-collapse collapse show"> <!-- Campos do formulário --> <input id="txtPonto" /> <input id="txtNome" /> <input id="txtRamal" /> <input id="ddtSetorRequisitante" /> <!-- DropDownTree AQUI --> </div> </div> <!-- Popup renderizado FORA do accordion --> <body> <div class="e-popup e-ddl-popup"> <!-- POPUP DO SYNCFUSION --> <ul> <li>Setor 1</li> <li>Setor 2</li> </ul> </div> </body>

Sequência de Eventos do Bug

  1. Usuário clica no DropDownTree → popup abre (renderizado no <body>)
  2. Usuário clica em um setor no popup → evento click dispara
  3. Bootstrap Collapse detecta clique "fora" do #collapseRequisitante
  4. Bootstrap automaticamente dispara evento hide.bs.collapse
  5. Accordion fecha (display: none)
  6. Campos são limpos (comportamento da aplicação)
Conclusão: O Bootstrap não consegue distinguir entre um "clique fora intencional para fechar" e um "clique em popup Syncfusion legítimo". Isso causava fechamento indesejado.
